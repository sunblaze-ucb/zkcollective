zkCollective: building a secure, universal foundation for multichain interoperability

Follow us on Twitter: @zkcollective


zkBridge is only the beginning.

More info: zkbridge.org, https://arxiv.org/pdf/2210.00264.pdf

Goals



1. Security

Trust-minimized
Defense in depth: proof-diversity, n-version programming, multiple implementations
Smart contract audit; zk circuit audit


2. Efficiency


3. User experience (low latency)

Potentially combining it with optimistic approach for shorter latency


4. Low fees

Close to 0 fees


5. Extensibility

Provide base layer; allow others to build rich functionalities on top


6. Universal

Help each chain to be zk friendly (proof-friendly)
Build out state proofs


7. Sustainability


zkCollective



Principles

  • Open source & open ecosystem
  • Public good
  • Collaborative



A collection of members (entities, projects, individuals) working together to achieve these goals, including:

  • Researchers
  • Developers
  • L1/L2
  • Bridges
  • Bridge aggregators
  • dApps
  • Node operators (for relayers and proof generators)
  • Investors
  • Users

zkCollective activities (in collaboration with members)

  • Research
  • Education
  • Build reference implementation & productization
  • Security audit
  • zk tool development
  • Standardization
  • New proposals to improve zk-friendliness
  • Build a self-sustainable ecosystem
